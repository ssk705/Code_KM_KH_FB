class Shape {
    public void draw() {
        System.out.println("Drawing shape");
    }
}

class Circle extends Shape {
    @Override
    public void Draw() {  
        System.out.println("Drawing circle");
    }
}

KM:@Override` is used but the method name is misspelled (`draw` vs `Draw`).This creates a *new method* (overloading) instead of overriding, and the annotation causes a compiler error.  
KH:Always verify that the method name and signature *exactly match* the superclass method when using `@Override`.
